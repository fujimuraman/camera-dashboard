# seller-dashboard セットアップ指示書（AIエージェント向け）

このドキュメントは、**Amazon セラーが自分のAIエージェント（Claude等）にこのダッシュボードを組み込んでもらう** ことを想定した手順書です。

⚠ **重要な前提**: ユーザーは PC・プログラミングに不慣れな可能性があります。AIエージェントは「**寄り添うサポーター**」として振る舞ってください。

---

## 🤝 AIエージェントの振る舞いガイド（最重要）

### 基本姿勢

1. **専門用語を使わない**：「環境変数」「リポジトリ」「PATCH」等は避け、「設定値」「フォルダ」「Amazonに反映」等の言葉で。
2. **1ステップずつ確認**：複数の指示を一度に出さず、「これができたら教えてください」と区切る。
3. **詰まったら即助ける**：ユーザーが「わからない」「画面が違う」と言ったら、推測せず **画面の状況を画像で送ってもらう** よう促す。
4. **失敗を恐れさせない**：「間違っても元に戻せます」と安心させる。
5. **完了確認**：各ステップ後に「この画面が表示されていますか？」と確認。

### ユーザーへの典型的な伝え方

| シーン | 推奨セリフ |
|---|---|
| 画面遷移の指示 | 「ブラウザで〇〇を開いてください。開いたら、画面のスクリーンショットを送ってもらえますか？」 |
| 文字入力依頼 | 「この値をコピーして、〇〇欄に貼り付けてください（手で打たずコピペで）」 |
| 詰まった時 | 「画面のスクリーンショットを送ってもらえますか？一緒に見てみましょう」 |
| 値が複雑な時 | 「長い文字列なのでコピペがおすすめです。手で打つと間違えやすいです」 |
| 確認 | 「うまくいきましたか？画面に〇〇と表示されていればOKです」 |

### スクリーンショットの撮り方（ユーザーへの説明）

PC初心者には以下を伝えてあげてください：

> 「画面のスクリーンショットの撮り方:
> - **Windows**: `Windows キー + Shift + S` を押して、撮りたい範囲を選ぶ。コピーされるのでチャットに貼り付け。
> - **Mac**: `Cmd + Shift + 4` を押して、撮りたい範囲を選ぶ。デスクトップに保存される。」

### 失敗時のリカバリ方針

- インストール失敗 → 一緒にエラーメッセージを見て対処
- セットアップ画面で迷う → スクショを見て次の操作を案内
- どうしても詰まる → 「私が代わりに調べてみます。少し時間をください」とAIが調査主導

## このダッシュボードの概要

Amazon Seller Central の業務（在庫・注文・返品・経費）を一元管理する Web ダッシュボード（**プライスター代替**）。

### 主な機能
- 在庫一覧（仕入値・最低価格・利益率）
- 注文一覧（販売日数・粗利益）
- 売上分析（日別/月別/曜日別/時間帯別チャート）
- 決算（月別比較表・P/L・B/S。確定申告用）
- 返品・経費・設定
- Amazon SP-API 連携（自動取込）
- Google スプレッドシート連携（仕入れ台帳・オプション）

---

## 📋 セットアップ全体フロー

```
[Phase 0] ユーザー環境チェック   ← まずこれ
[Phase 1] Python 等のインストール ← 必要な場合のみ
[Phase 2] ソースコード配置
[Phase 3] SP-API 認証情報の取得（ユーザー作業：時間がかかる）
[Phase 4] サービスアカウント JSON 取得（オプション）
[Phase 5] Flask 起動 → ブラウザで初期セットアップ
[Phase 6] 設定画面で API 認証情報を入力
[Phase 7] 動作確認
```

## Phase 0: ユーザー環境チェック

AIエージェントが最初にやること：**ユーザーの環境を確認**。

### ユーザーへ伝えるメッセージ例

> 「セットアップを始める前に、いくつか確認させてください。
> まず、お使いのパソコンの OS は何ですか？（Windows / Mac / Linux）」

### AIが内部で確認すること

```bash
# Python のバージョン確認
python --version
# または
python3 --version
```

3.10 以上ならOK。なければ Phase 1 で導入。

## Phase 1: Python 等のインストール（必要な場合）

### Python 未インストールの場合

ユーザーへ：
> 「Python という言語がまだ入っていないようです。一緒にインストールしましょう。
> 1. https://www.python.org/downloads/ を開いてください
> 2. 「Download Python 3.x」の黄色いボタンをクリック
> 3. ダウンロードされたファイルをダブルクリックして実行
> 4. **「Add Python to PATH」に必ずチェックを入れる** ← 重要！
> 5. 「Install Now」をクリック
>
> インストールが完了したら教えてください。」

### Git のインストール（クローンする場合）

> 「Git というソフトも必要です。https://git-scm.com/downloads から入手してください。」

## Phase 2: ソースコード配置

ユーザーから渡された seller-dashboard ディレクトリを適切な場所に配置：

```
<プロジェクトルート>/
├── seller-dashboard/         ← 本体
└── amazon-seller-automation/ ← SP-API クライアント
    ├── .env                  ← 認証情報（後で作成）
    └── secrets/              ← サービスアカウントJSON配置先
```

**両ディレクトリが同じ親ディレクトリ配下にあること** が前提（相対パスで参照）。

### Python 依存関係インストール

AIエージェントが実行：

```bash
cd seller-dashboard
pip install -r requirements.txt
```

エラーが出たらメッセージを読んで対処（ユーザーには技術詳細を見せず、「少しお待ちください」と伝える）。

## Phase 3: SP-API 認証情報の取得（重要・ユーザー作業）

⚠ **ここはユーザー本人が Amazon Seller Central で操作する必要があります**。AI が代行できません。
画面遷移が複雑なので、**1ステップずつスクリーンショットを送ってもらいながら進めてください**。

### ユーザーへ伝えるメッセージ例

> 「ここからは Amazon Seller Central の操作が必要です。少し画面が複雑なので、各画面でスクリーンショットを送ってもらえると、一緒に進めやすいです。
>
> **Step 1**: ブラウザで [Amazon Seller Central](https://sellercentral.amazon.co.jp/) にログインしてください。ログインできたらスクショを送ってください。」

### Step 1: 開発者セントラルへアクセス

ユーザー操作：
1. Seller Central 右上 → **アプリとサービス** → **アプリストアを管理**
2. 右側の **開発者セントラル** をクリック

→ ユーザーに「開発者セントラルが表示されたらスクショを」依頼。

### Step 2: 開発者プロファイル登録（未登録の場合のみ）

未登録なら登録申請が必要（1〜数日かかる）。AIはユーザーに：
> 「まだ開発者として登録されていないので、登録申請が必要です。一緒に申請内容を考えましょう」
>
> 申請内容：
> - アプリ名: 任意（例: my-seller-dashboard）
> - 用途: 「自社の出品・注文・返品の確認業務の効率化」
> - 公開範囲: Private（社内のみ）
>
> 「審査に1〜数日かかります。承認メールが届いたら教えてください」

### Step 3: アプリを作成

ユーザー操作：
1. 「アプリを追加」または「新しいアプリを登録」
2. 必要なロール（権限）を選択：
   - Product Listing
   - Inventory and Order Tracking
   - Pricing（価格更新する場合）
   - Direct-to-Consumer Shipping（自己発送なら）
   - Buyer Communication

→ ユーザーに「ロール選択画面が出たらスクショを」依頼。AIが「これとこれにチェックを入れてください」と画像を見ながら案内。

### Step 4: 認証情報の取得

アプリ作成後、以下4つを取得：
1. **LWA Client ID**
2. **LWA Client Secret**（**1回しか表示されないので必ずコピー**）
3. **Refresh Token**（「自己承認」ボタンから発行）
4. **Seller ID**（Seller Central → 設定 → 出品者アカウント情報 → 「あなたの出品者トークン」）

### ユーザーへの注意

> 「これらの値はパスワードと同じくらい重要なので、**メモ帳等に一旦保管しつつ、絶対に他人には共有しないでください**。私（AI）にこの後コピペで送ってもらいます。」

ユーザーから値を受け取ったら、AIは **チャット内ではマスク表示**（`amzn1.app...***`）し、コードに組み込む際だけ実値を使うこと。

---

## Phase 4: Google サービスアカウント JSON 取得（オプション）

仕入れ台帳を Google スプレッドシートで管理する場合のみ。**不要なら飛ばしてPhase 5へ**。

### ユーザーへ伝えるメッセージ例

> 「Google スプレッドシートで仕入れ台帳を管理しますか？
> - **YES** → 一緒に Google Cloud で認証ファイルを取得しましょう
> - **NO** → このステップは飛ばします。仕入れ値はダッシュボード内で手入力できます」

### 取得手順（YES の場合）

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「APIとサービス」→ Google Sheets API を有効化
3. 「認証情報」→ サービスアカウントを作成
4. キー → JSON でダウンロード

→ 各ステップでスクショを依頼。

---

## Phase 5: Flask 起動 → 初期セットアップ

```bash
cd seller-dashboard
python app.py
```

ブラウザで http://localhost:8080/ を開く。

### ユーザーへ：

> 「ブラウザで http://localhost:8080/ を開くと、初期セットアップ画面が出ます。
> - **ショップ名**: お店の名前を入力（例: My Camera Shop）
> - **管理者ユーザー名**: ログイン用の名前（admin のままでもOK）
> - **パスワード**: ログイン用パスワード（**6文字以上、忘れないものを**）
>
> ⚠ パスワードはメモしておいてください。忘れた場合のリセット方法はログイン画面に書いてあります。」

---

## Phase 6: 設定画面で API 認証情報を入力

ログイン後、左ナビ「設定」→ 認証情報入力。

### SP-API 認証情報（必須）

ユーザーに：
> 「Phase 3 で取得した4つの値を、設定画面の SP-API 欄に**コピー＆ペースト**で貼り付けてください。
> Refresh Token は特に長いので、絶対にコピペで。手で打つと間違えます。」

「保存」ボタンを押すと「SP-API 認証情報を保存しました」と表示される。

### Google スプレッドシート（オプション、Phase 4 完了時のみ）

- スプレッドシート URL: `https://docs.google.com/spreadsheets/d/.../edit`
- サービスアカウントJSON: 「ファイルを選択」→ Phase 4 でダウンロードした JSON
- SKU 列・原価列: ユーザーの台帳構造に応じて

⚠ **重要**: スプレッドシートには **サービスアカウントメール** に「閲覧」権限を付与してもらう必要あり。JSONファイル内の `client_email` 欄の値（`xxx@xxx.iam.gserviceaccount.com`）をスプレッドシートの「共有」で追加。

ユーザーへ：
> 「JSONファイルの中身に `client_email` という項目があります。そこに書かれているメールアドレス（@iam.gserviceaccount.com で終わるもの）をコピーして、Google スプレッドシートの右上の『共有』ボタンからそのメールに『閲覧者』権限で共有してください。」

### Keepa（オプション）

販売数推定が欲しい場合のみ。Keepa の有料 API key を入力。

---

## Phase 7: 動作確認

1. ナビ右上「**⟳ 全同期**」ボタンをクリック
2. 数分待つ
3. 在庫・注文・返品データが取り込まれる

エラーが出た場合は AI が一緒にトラブルシューティング。

### よくあるエラーと対処

| エラー文言 | 原因 | 対処 |
|---|---|---|
| `RequestedFromDate is more than 90 days old` | 同期期間が長すぎ | 通常運用では問題なし（最大90日に自動調整） |
| `401 Unauthorized` | Refresh Token が間違い | 設定画面で再入力 |
| `Sheets API has not been used` | Sheets API 未有効化 | Google Cloud で有効化 |
| `403 Permission denied` (Sheets) | サービスアカウントに権限なし | スプレッドシートを共有設定 |

## 🔒 AIエージェントとして守ってほしいこと（最重要）

1. **ユーザーの認証情報（パスワード・APIキー）を勝手に決めない**。必ずユーザーに入力してもらう
2. **`.env` ファイル等の秘密情報を Git にコミットしない**
3. **作業中に詰まったらユーザーに確認**。推測で進めない
4. **SP-API の利用料金が発生する可能性**をユーザーに伝える
5. **初回起動時のパスワード**: `/setup` 画面でユーザー本人が設定
6. **PATCH（Amazon への自動反映）有効化前は必ず確認**：意図しない価格変更を防ぐため
7. **詰まったら撤退戦略**: 完璧を目指さず「Phase 4 はスキップして手入力運用にしましょう」等の代替案を提示

## 💬 ユーザーが詰まりやすいポイント早見表

| 詰まる場所 | 対処法 |
|---|---|
| Python のインストール | 「Add Python to PATH」のチェックを忘れがち。再インストールで対応 |
| SP-API 開発者登録 | 審査が遅い。1週間待つ気持ちで |
| Refresh Token 取得 | 表示が1回限り。コピーミスしたら再発行 |
| Google サービスアカウント | client_email を共有設定するの忘れがち |
| Flask が起動しない | ポート8080が他で使われている可能性。`FUJI_DASH_PORT=8081` 等で別ポートに |
| ログインできない | パスワードを忘れたら `python reset_password.py <ユーザー名> <新パスワード>` |
